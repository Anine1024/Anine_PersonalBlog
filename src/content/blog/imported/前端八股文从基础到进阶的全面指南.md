## JavaScript核心八股文

### 1. instanceof 原理与实现

`instanceof` 是JavaScript中检测对象类型的常用运算符，其核心原理是：**检查构造函数的 `prototype` 属性是否出现在实例对象的原型链上**。

```javascript
// 手写 instanceof
function myInstanceof(left, right) {
  // 获取对象的原型
  let proto = Object.getPrototypeOf(left);
  // 获取构造函数的 prototype
  const prototype = right.prototype;
  
  while (true) {
    if (proto === null) return false;
    if (proto === prototype) return true;
    proto = Object.getPrototypeOf(proto);
  }
}

// 测试
function Person(name) {
  this.name = name;
}
const p = new Person('Alice');
console.log(myInstanceof(p, Person)); // true
console.log(myInstanceof(p, Object)); // true
console.log(myInstanceof(p, Array));  // false
```

**关键点**：
- 通过 `Object.getPrototypeOf()` 获取原型链
- 循环遍历直到找到匹配或到达原型链末端
- 注意基本类型（如 `'hello' instanceof String` 返回 `false`）

### 2. for...in 与普通 for 循环的区别

很多开发者会误用 `for...in` 遍历数组，这可能导致意想不到的问题：

```javascript
const arr = [10, 20, 30];
arr.customProp = 'test';

// 普通 for 循环
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]); // 10, 20, 30
}

// for...in 循环
for (let key in arr) {
  console.log(key, arr[key]); 
  // 0 10, 1 20, 2 30, customProp test
}
```

**核心区别**：
- `for...in` 遍历所有可枚举属性（包括原型链上的）
- 普通 `for` 循环只遍历数组索引
- 数组遍历推荐使用 `for...of` 或 `forEach`

```javascript
// 推荐方式
for (let item of arr) {
  console.log(item); // 10, 20, 30
}

arr.forEach((item, index) => {
  console.log(item, index);
});
```

### 3. src 与 href 的区别

在HTML中，`src` 和 `href` 虽然都用于加载外部资源，但行为完全不同：

```html
<!-- href：超链接引用，不会阻塞页面解析 -->
<link href="styles.css" rel="stylesheet">
<a href="https://example.com">链接</a>

<!-- src：资源嵌入，会阻塞页面渲染 -->
<script src="app.js"></script>
<img src="logo.png" alt="logo">
```

**关键差异**：
- `href` 建立文档与资源的关联，浏览器会并行加载
- `src` 将资源内容嵌入当前文档位置，加载时会暂停其他处理
- 这也是为什么建议将 `<script>` 放在页面底部的原因

## Vue 核心八股文

### 1. MVVM 模式详解

MVVM（Model-View-ViewModel）是Vue的架构基础：

```javascript
// Model：数据模型
const model = {
  name: 'Alice',
  age: 25
};

// ViewModel：Vue实例（核心）
const vm = new Vue({
  data: model,
  methods: {
    updateName(newName) {
      this.name = newName;
    }
  }
});

// View：模板（声明式渲染）
// <div>{{ name }}</div>
```

**核心机制**：
- **Model**：纯JavaScript数据对象
- **View**：DOM模板（声明式）
- **ViewModel**：Vue实例，负责数据绑定和指令解析

**优势**：
- 数据驱动视图，无需手动操作DOM
- 视图与逻辑分离，提高可维护性
- 支持双向绑定（如 `v-model`）

### 2. Vue 响应式原理

Vue 2.x 使用 `Object.defineProperty` 实现响应式：

```javascript
// 简化版响应式实现
function defineReactive(obj, key, val) {
  // 递归处理嵌套对象
  observe(val);
  
  Object.defineProperty(obj, key, {
    get() {
      console.log(`读取 ${key}: ${val}`);
      return val;
    },
    set(newVal) {
      if (newVal !== val) {
        console.log(`设置 ${key}: ${newVal}`);
        val = newVal;
        // 触发更新...
      }
    }
  });
}

function observe(obj) {
  if (typeof obj !== 'object' || obj === null) return;
  for (let key in obj) {
    defineReactive(obj, key, obj[key]);
  }
}

// 使用
const data = { name: 'Alice', info: { age: 25 } };
observe(data);
data.name = 'Bob'; // 触发 set
console.log(data.name); // 触发 get
```

**Vue 3 升级**：使用 `Proxy` 解决 `Object.defineProperty` 的局限性：
- 支持数组索引和长度变化
- 无需递归遍历所有属性
- 性能更好，代码更简洁

## 性能优化八股文

### 1. 图片懒加载原理

图片懒加载是前端性能优化的重要手段，核心思想是：**只加载当前可视区域的图片**。

```javascript
// 基于 IntersectionObserver 的实现
function lazyLoadImages() {
  const images = document.querySelectorAll('img[data-src]');
  
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        const img = entry.target;
        img.src = img.dataset.src;
        img.removeAttribute('data-src');
        observer.unobserve(img);
      }
    });
  });
  
  images.forEach(img => observer.observe(img));
}

// HTML 使用
// <img data-src="real-image.jpg" src="placeholder.jpg" alt="lazy">
```

**传统实现**（scroll 事件监听）：
```javascript
function lazyLoadScroll() {
  const images = document.querySelectorAll('img[data-src]');
  
  function checkImages() {
    images.forEach(img => {
      const rect = img.getBoundingClientRect();
      if (rect.top < window.innerHeight && rect.bottom > 0) {
        img.src = img.dataset.src;
        img.removeAttribute('data-src');
      }
    });
  }
  
  window.addEventListener('scroll', checkImages);
  checkImages(); // 初始检查
}
```

**优化要点**：
- 使用 `IntersectionObserver` 替代 scroll 事件，性能更好
- 设置占位图或骨架屏，提升用户体验
- 结合 `loading="lazy"` 属性（现代浏览器原生支持）

### 2. 防抖与节流

防抖和节流是控制高频事件触发频率的经典技术：

```javascript
// 防抖：等待用户停止操作后才执行
function debounce(fn, delay = 300) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流：限制执行频率（每段时间内只执行一次）
function throttle(fn, interval = 200) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

// 使用场景
const handleSearch = debounce((query) => {
  console.log('搜索:', query);
}, 500);

const handleScroll = throttle(() => {
  console.log('滚动事件');
}, 200);
```

**适用场景**：
- 防抖：搜索框输入、窗口resize
- 节流：滚动加载、按钮点击

## 工程化八股文

### 1. Webpack 构建流程

Webpack 的构建流程可以概括为四个核心步骤：

```javascript
// webpack.config.js 简化示例
module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js'
  },
  module: {
    rules: [
      { test: /\.js$/, use: 'babel-loader' },
      { test: /\.css$/, use: ['style-loader', 'css-loader'] }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({ template: './src/index.html' })
  ]
};
```

**构建流程详解**：

1. **初始化参数**：合并配置文件与Shell参数，创建 `Compiler` 对象
2. **开始编译**：调用 `Compiler.run()`，加载所有插件
3. **编译模块**：从入口文件开始，递归解析依赖，使用 Loader 转换文件
4. **输出资源**：根据依赖关系生成 Chunk，输出到指定目录

```javascript
// 简化版流程模拟
class Compiler {
  constructor(config) {
    this.config = config;
    this.hooks = {};
  }
  
  run() {
    // 1. 读取入口
    const entry = this.config.entry;
    
    // 2. 解析依赖（递归）
    const modules = this.parseModule(entry);
    
    // 3. 生成代码
    const outputCode = this.generateCode(modules);
    
    // 4. 输出文件
    this.emitFile(outputCode);
  }
  
  parseModule(filePath) {
    // 读取文件内容
    let source = fs.readFileSync(filePath, 'utf-8');
    
    // 应用 Loader
    this.config.module.rules.forEach(rule => {
      if (rule.test.test(filePath)) {
        source = rule.use.reduce((code, loader) => {
          return require(loader)(code);
        }, source);
      }
    });
    
    // 递归解析依赖...
    return { filePath, source, dependencies: [] };
  }
}
```

**关键概念**：
- **Loader**：文件转换器（如 babel-loader）
- **Plugin**：生命周期钩子（如 HtmlWebpackPlugin）
- **Chunk**：代码块，对应输出文件

### 2. 模块化规范对比

前端模块化发展经历了多个阶段，理解它们的差异很重要：

```javascript
// CommonJS (Node.js)
const fs = require('fs');
module.exports = { readFile: fs.readFile };

// ES Module (现代浏览器)
import fs from 'fs';
export function readFile(path) {
  return fs.readFileSync(path);
}

// AMD (RequireJS)
define(['jquery'], function($) {
  return { getElement: (id) => $(`#${id}`) };
});

// UMD (通用模块)
(function(root, factory) {
  if (typeof define === 'function' && define.amd) {
    define(['jquery'], factory);
  } else if (typeof module === 'object' && module.exports) {
    module.exports = factory(require('jquery'));
  } else {
    root.myModule = factory(root.jQuery);
  }
}(this, function($) {
  return { log: (msg) => console.log(msg) };
}));
```

**核心差异**：
- CommonJS：同步加载，适合服务端
- ES Module：静态分析，支持 tree-shaking
- AMD：异步加载，适合浏览器
- UMD：兼容多种环境
