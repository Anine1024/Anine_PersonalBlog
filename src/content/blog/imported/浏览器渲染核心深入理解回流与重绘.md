## 引言：为什么你的页面会“卡顿”？

在日常开发中，你是否遇到过这样的场景：一个简单的列表渲染，页面却出现了明显的闪烁或卡顿；修改了一个元素的背景色，整个页面都跟着“抖”了一下？这背后的元凶，往往就是浏览器的**回流（Reflow）**和**重绘（Repaint）**。

作为前端开发者，理解这两个概念不仅是应付面试的“八股文”，更是写出高性能、流畅交互页面的基本功。本文将从浏览器渲染流程入手，深入剖析回流与重绘的底层原理、触发条件，并给出经过实践检验的优化方案。读完这篇文章，你将能清晰地回答：什么操作会触发回流？回流和重绘谁更昂贵？以及如何系统地减少它们的发生。

## 一、先搞懂：浏览器是如何画出一张页面的？

在讨论回流与重绘之前，我们必须先理解浏览器的**渲染流水线**。这个过程可以简化为以下几个关键步骤：

1.  **解析HTML，构建DOM树**：浏览器将HTML字节流解析成节点对象，形成文档对象模型（DOM Tree）。
2.  **解析CSS，构建CSSOM树**：同时解析CSS样式（包括内联、外部样式表和`<style>`标签），生成层叠样式表对象模型（CSSOM Tree）。
3.  **合并生成渲染树（Render Tree）**：将DOM树与CSSOM树合并，只包含**可见节点**（例如，`display: none`的节点不会被包含，但`visibility: hidden`的节点会包含，因为它占据空间且需要绘制）。
4.  **布局（Layout）**：计算渲染树中每个节点的**几何信息**（位置、大小、边距等）。这其实就是**回流（Reflow）**。
5.  **绘制（Paint）**：将布局阶段计算出的具体像素填充到屏幕上，比如绘制文本、颜色、边框、阴影等。这就是**重绘（Repaint）**。
6.  **合成（Composite）**：将不同的绘制层合并成一个最终的图像，然后显示在屏幕上。现代浏览器（如Chrome）会利用GPU加速合成层，提升性能。

**核心要点**：**回流一定会触发重绘，但重绘不一定触发回流**。因为如果只改变颜色（重绘），元素的几何尺寸没变，就不需要重新布局。

## 二、回流（Reflow）：最昂贵的性能杀手

### 2.1 什么是回流？

回流，也叫**重排（Layout）**，是指浏览器为了重新计算页面上部分或全部元素的几何属性（位置、尺寸）而进行的过程。由于页面布局是“流式”的，一个元素的尺寸变化，可能像多米诺骨牌一样，引发其子元素、父元素甚至兄弟元素的连锁重排。

### 2.2 哪些操作会触发回流？

以下操作几乎是“必发”回流的：

- **添加或删除可见的DOM元素**：比如`appendChild`或`removeChild`。
- **元素尺寸改变**：`width`、`height`、`padding`、`border`、`margin`等。
- **元素位置改变**：`top`、`left`、`position`（从`static`变为`absolute`等）。
- **内容改变**：文本内容变化、图片尺寸变化（图片加载完成后宽高未知）。
- **浏览器窗口尺寸改变**：`resize`事件。
- **获取某些“立即生效”的布局属性**：这是新手最容易踩的坑。当你读取以下属性时，浏览器为了给你“最新最准确”的值，会强制触发回流（即使之前有批量的样式修改被缓存）：
    - `offsetTop`、`offsetLeft`、`offsetWidth`、`offsetHeight`
    - `scrollTop`、`scrollLeft`、`scrollWidth`、`scrollHeight`
    - `clientTop`、`clientLeft`、`clientWidth`、`clientHeight`
    - `getComputedStyle()` 或 `currentStyle`

**代码示例：强制回流的陷阱**

```javascript
// 错误示范：在循环中反复读取布局属性，导致频繁回流
const box = document.getElementById('box');
for (let i = 0; i < 1000; i++) {
    // 每次读取 offsetTop 都会强制触发一次回流
    console.log(box.offsetTop);
    box.style.left = i + 'px'; // 修改也触发回流
}

// 正确做法：先批量读取，再批量写入
const box = document.getElementById('box');
const topValue = box.offsetTop; // 只触发一次回流读取
box.style.left = '100px'; // 写入，可能触发一次回流
box.style.top = '200px'; // 合并到同一批
```

## 三、重绘（Repaint）：成本也不低

### 3.1 什么是重绘？

重绘发生在元素**外观发生变化**，但**几何属性不变**时。比如修改`color`、`background-color`、`visibility`、`outline`、`box-shadow`等。浏览器只需要重新绘制像素，不需要重新计算布局。

### 3.2 重绘的触发场景

- 修改颜色相关属性：`color`、`background-color`、`border-color`。
- 修改可见性：`visibility`、`opacity`（注意：`opacity`在非合成层下会触发重绘，在合成层下则不会）。
- 修改背景图片、背景位置等。
- 修改文本样式：`text-decoration`、`font-weight`（但注意：`font-size`会触发回流）。

**代码示例：重绘与回流的区别**

```css
/* 这个操作只触发重绘 */
.element {
    background-color: red;
}

/* 这个操作同时触发回流和重绘 */
.element {
    width: 200px;
}
```

**性能对比**：回流的代价远高于重绘。回流需要重新计算布局树中所有受影响节点的几何信息（可能涉及整个子树甚至整个文档），而重绘只需要将已有的像素重新填充。在复杂页面中，一次回流可能耗时几十甚至几百毫秒，而重绘通常在几毫秒内完成。

## 四、性能优化：如何让浏览器“少干活”？

理解了原理，优化就变得有迹可循。核心思想是：**减少触发次数、降低影响范围、利用浏览器批处理机制**。

### 4.1 批量修改样式（利用浏览器的渲染队列）

浏览器很聪明，它会将连续的样式修改放入一个“渲染队列”，等到队列满了或者遇到强制读取布局属性的操作时，才会一次性执行回流。我们要做的就是**避免打断这个队列**。

```javascript
// 坏做法：三次修改，大概率触发三次回流
const el = document.getElementById('myEl');
el.style.width = '100px';
el.style.height = '100px';
el.style.margin = '10px';

// 好做法：合并修改
el.style.cssText = 'width: 100px; height: 100px; margin: 10px;';

// 或者使用 class 切换
el.classList.add('my-class');
```

### 4.2 使用 `requestAnimationFrame` 进行动画

`requestAnimationFrame` 会把你的动画回调安排在下一次浏览器重绘之前执行，它会自动将多次DOM操作合并到同一帧中，避免不必要的回流。

```javascript
// 坏做法：使用 setInterval 直接修改
let left = 0;
setInterval(() => {
    box.style.left = left + 'px';
    left++;
}, 16); // 约60fps

// 好做法：使用 rAF
function animate() {
    box.style.left = left + 'px';
    left++;
    requestAnimationFrame(animate);
}
requestAnimationFrame(animate);
```

### 4.3 将元素“离线”操作

对于复杂的DOM操作（比如在一个列表里插入1000个元素），可以先让元素脱离文档流，修改完成后再放回去。这样只会触发**两次回流**（脱离和插入）。

```javascript
// 方法1：隐藏元素 -> 修改 -> 显示
const ul = document.getElementById('list');
ul.style.display = 'none'; // 触发一次回流
// 这里进行大量的 appendChild 操作
for (let i = 0; i < 1000; i++) {
    const li = document.createElement('li');
    li.textContent = `Item ${i}`;
    ul.appendChild(li);
}
ul.style.display = 'block'; // 触发一次回流

// 方法2：使用 DocumentFragment（更优雅）
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
    const li = document.createElement('li');
    li.textContent = `Item ${i}`;
    fragment.appendChild(li);
}
ul.appendChild(fragment); // 只触发一次回流
```

### 4.4 利用合成层（Composite Layer）进行动画

现代浏览器（Chrome、Safari）支持将某些元素提升为单独的合成层。合成层上的动画（如`transform`、`opacity`）**不会触发回流和重绘**，而是由GPU直接处理，性能极高。

```css
/* 触发合成层的常见属性 */
.animated-element {
    will-change: transform; /* 告诉浏览器我要变了，提前准备 */
    /* 或者直接使用 transform 进行动画 */
    transition: transform 0.3s ease;
}

/* 注意：不要滥用 will-change，会消耗大量内存 */
```

**何时使用**：对于频繁动画的元素（如轮播图、弹窗、滚动列表），使用`transform: translateZ(0)`或`will-change: transform`将其提升到合成层，可以避免90%以上的布局抖动。

## 总结

| 特性 | 回流（Reflow） | 重绘（Repaint） |
|------|---------------|----------------|
| **触发条件** | 几何属性改变（尺寸、位置、增删DOM） | 外观属性改变（颜色、背景、阴影） |
| **性能开销** | 非常高（计算布局树） | 较低（填充像素） |
| **是否必然** | 回流必然导致重绘 | 重绘不一定需要回流 |
| **关键优化** | 批量读写、离线DOM、使用合成层 | 减少不必要的样式修改 |

**一句话记住**：**回流是“动筋骨”，重绘是“换衣服”**。写代码时，优先考虑通过`transform`、`opacity`来实现动画效果，并始终警惕在循环中读取布局属性。掌握这些，你的页面性能将提升一个台阶，面试官也会对你刮目相看。
